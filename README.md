# NpcNotebook — Блокнот NPC

Desktop-приложение для мастера **Dungeons & Dragons**: храните персонажей (NPC), группируйте их по городам, фракциям и организациям, ведите карточки с портретами и заранее заготовленными репликами — всё в одном файле.

Часть экосистемы утилит для мастера; визуальный стиль и подход к справке совпадают с **DnDMapHelper** (карта приключений).

## Возможности

- **Группы персонажей** — произвольные категории: город, гильдия, таверна и т.д.
- **Список NPC** — портрет и имя; поиск по имени, расе и роду деятельности.
- **Карточка персонажа** — имя, раса, возраст, род деятельности, характер, загружаемый портрет.
- **Отношения** — связи с другими персонажами из блокнота (несколько записей на одного NPC).
- **Диалоги и реплики** — именованные вкладки с текстом для игры за столом.
- **Drag & drop** — перетащите персонажа на группу, чтобы включить его в неё.
- **Масштабируемый интерфейс** — разделители между списком, карточкой и блоком диалогов.
- **Сохранение в один файл** — формат `.npcbook` (данные + портреты).
- **Встроенная справка** — кнопка «?» в окне программы.

## Требования

- **ОС:** Windows 10 или новее
- **.NET SDK 10.0** (или новее) — для сборки из исходников
- **.NET Desktop Runtime 10.0** — для запуска собранного приложения

## Быстрый старт

### Сборка и запуск из исходников

```bash
git clone <URL-репозитория>
cd NpcNotebook
dotnet build NpcNotebook.slnx
dotnet run --project NpcNotebook/NpcNotebook.csproj
```

### Публикация (один exe)

```bash
dotnet publish NpcNotebook/NpcNotebook.csproj -c Release -r win-x64 --self-contained false
```

Готовый файл будет в `NpcNotebook/bin/Release/net10.0-windows/win-x64/publish/`.

## Как пользоваться

1. **Открыть** — загрузить блокнот `.npcbook`.
2. **Сохранить** — записать блокнот; откроется диалог выбора пути и имени файла.
3. **+ Группа** — создать группу (например, «Waterdeep» или «Гильдия воров»).
4. **+ Персонаж** — новый NPC; клик по нему в списке открывает карточку справа.
5. Перетащите персонажа на заголовок группы — он попадёт в эту группу.

Один персонаж может состоять в **нескольких группах** одновременно. Персонажи без группы отображаются в разделе **«Без группы»**.

Подробная инструкция — в справке внутри программы (кнопка **?**).

## Формат `.npcbook`

Файл блокнота — ZIP-архив:

| Содержимое | Описание |
|------------|----------|
| `notebook.json` | группы, персонажи, отношения, вкладки диалогов |
| `portraits/` | изображения портретов (PNG, JPG и др.) |

Все данные кампании и портреты хранятся в одном месте — удобно копировать и делать резервные копии.

## Структура проекта

```
NpcNotebook/
├── Models/           # Модели данных и DTO для сохранения
├── Services/         # Сессия приложения и работа с .npcbook
├── ViewModels/       # MVVM (CommunityToolkit.Mvvm)
├── Views/            # Окна и XAML
├── Help/             # Текст встроенной справки
├── Themes/           # Fantasy-тема интерфейса
└── Converters/       # Конвертеры для привязок WPF
```

**Стек:** WPF, .NET 10, MVVM, JSON + ZIP для persistence.
